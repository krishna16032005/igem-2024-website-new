<h2 id="tables">Tables</h2>

<p>For tables there are two different syntaxes:</p>

<ol>
<li>GitHub Markdown Style</li>
</ol>

<div class="codehilite">
<pre><span></span><code>| Header 1 | Header 2 |
| -------- | -------- |
| Cell 1 | Cell 2 |
| Cell 3 | Cell 4 |
</code></pre>
</div>

<table>
<thead>
<tr>
<th>Header 1</th>
<th>Header 2</th>
</tr>
</thead>
<tbody>
<tr>
<td>Cell 1</td>
<td>Cell 2</td>
</tr>
<tr>
<td>Cell 3</td>
<td>Cell 4</td>
</tr>
</tbody>
</table>

<ol start="2">
<li>Google Wiki Style</li>
</ol>

<div class="codehilite">
<pre><span></span><code>|| <span class="gs">**Header 1**</span> || <span class="gs">**Header 2**</span> ||
|| Cell 1 || Cell 2 ||
|| Cell 3 || Cell 4 ||
</code></pre>
</div>

<table>
<tbody>
<tr>
<td><strong>Header 1</strong></td>
<td><strong>Header 2</strong></td>
</tr>
<tr>
<td>Cell 1</td>
<td>Cell 2</td>
</tr>
<tr>
<td>Cell 3</td>
<td>Cell 4</td>
</tr>
</tbody>
</table>

<p>Both produce similar results. In the Github style, the row above <code>-----</code> is in the <code>&lt;thead&gt; &lt;/thead&gt;</code> tags. But in the Google Wiki style, all rows are in <code>&lt;tr&gt; &lt;/tr&gt;</code> tags.</p>
