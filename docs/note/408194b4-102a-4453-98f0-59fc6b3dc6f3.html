<ol start="6">
<li>If the files you've received match the above structure, then the cloning has been done correctly. I am also including the names of the migration files. Make sure you have all of them.</li>
<li>Now, you should start a python development environment using <code>python -m venv .venv</code>. You should first run <code>pip3 install virtualenv</code> to install virtualenv and create virtual environment. Then run <code>.venv/Scripts/activate</code> to activate the virtual environment. (After you are done editing or testing the code, just run <code>deactivate</code> to exit the virtual environment)</li>
</ol>

<blockquote>
<p>[!IMPORTANT]
If you are in Windows and have not used python virtual environment before, chances are you won't be able to run the above command. To fix this, open PowerShell as an administrator and run <code>Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser</code>. This will allow running of scripts (which has been digitally signed by a trusted publisher) on your computer.</p>
</blockquote>

<ol start="8">
<li>Run <code>pip3 install -r requirements.txt</code> to install all the dependencies required.</li>
<li><code>cd</code> into the <code>wiki</code> folder and run <code>python manage.py makemigrations notebook</code> followed by <code>python manage.py migrate</code>. This will set up the database for the website.</li>
<li>Now run <code>python manage.py runserver</code> to start the development server if you want to see the website. Then go to <code>http://127.0.0.1:8000/docs</code> to visit the notebook page.</li>
</ol>

<p>If everything works fine, the installation is successful. If you wish to make Notes on the notebook app, you first need to create an user.</p>

<ol start="11">
<li>Shut the server down and run <code>python manage.py createsuperuser</code>. Give a username and a passoword. You may skip the email ID part by pressing <code>Enter</code> when prompted for the email ID.</li>
<li>Once the superuser is created, restart the server and go to <code>/admin</code>. Login using the superuser. Then go to Users. Click on the username of your superuser. Scroll down and enable the verified option. You should also assign a Role to the user. You may complete the rest of your profile if you wish but it is not mandatory (as of now). Then create some Departments and add the user in the member list of a Department.</li>
</ol>

<blockquote>
<p>[!IMPORTANT]
The Notes app might be a big buggy right now.</p>
</blockquote>

<ol start="13">
<li>Log out and go to <code>/login</code>. Login with the superuser account. Go to <code>/new</code> and create a note.</li>
<li>You can also use another account to create the notes. Go to <code>/signup</code> and creating an account. Then logout and go to <code>/admin</code> and signup with the superuser account to verify the new user. Assign them a Department and a position and save. Then login again with the new user and go to <code>/new</code> to create a note.</li>
</ol>

<h2 id="the-file-structure">The file structure</h2>

<ul>
<li>The files inside <code>wiki</code> directory:
<ul>
<li>The <code>notebook</code> folder contains the files for the notebook app. </li>
<li>The <code>static</code> folder contains the static files used by all the pages in the website (like fonts, common css, common JS, etc).</li>
<li>The <code>templates</code> folder contains <code>.html</code> files used by all the sites. It is the basic layout of the website.</li>
<li>The <code>wiki</code> folder contains all the configuration files of the django project.</li>
<li><code>.gitignore</code> contains names of all the folders that we don't want git to push to the online repo.</li>
<li><code>manage.py</code> is a utility file for any django project. It contains various housekeeping commands like <code>runserver</code> as used before.</li>
</ul></li>
<li><code>README.md</code> contains the markdown file for this text.</li>
<li><code>requirements.txt</code> lists all the dependencies required for this project.</li>
</ul>

<h2 id="contributing">Contributing</h2>

<p>Please follow the steps to contribute to this project:</p>

<ol>
<li>After you <code>git clone</code> the forked repository (proper forking described in step 1 of <a rel="noopener" target="_blank" href="https://github.com/nathaishik/igem-2024-website/tree/dev?tab=readme-ov-file#installation-for-development">Installation for Development</a>), switch to the 'dev' branch using <code>git checkout dev</code>. Then, run <code>git checkout -b USERNAME</code> to create a branch.</li>
</ol>

<blockquote>
<p>[!IMPORTANT]
You must follow step 1.</p>
</blockquote>

<ol start="2">
<li>Now, move to that branch using <code>git checkout USERNAME</code>.</li>
<li>Write and edit the code.</li>
<li>Run <code>python manage.py test</code> inside <code>wiki</code> directory to test your code. If you feel the need to introduce any new test for the feature that you've developed, write the test in the appropriate test file. Make sure that your code passes all tests.</li>
<li>After you are done, run <code>git add .</code> to add any new files that you might have created. You can skip this step if you haven't added anything new.</li>
<li>Run <code>git commit -am "COMMIT MESSAGE"</code> where COMMIT MESSAGE should mention what you modified.</li>
<li>Now this step is a bit of a trick. By default, you won't be able to push from this branch directly. But run <code>git push</code>. You should see an error message and that message contains the correct command to use to push to the online repository. Copy, paste and run that command to push your changes.</li>
<li>Open the repository from your browser. Chances are, you should see a banner asking you to create a Pull Request. If not, manually create a pull request by navigating to the appropriate tab. Fill in the necessary details. Tt is important that you ask for merging in the <code>dev</code> branch. Then create the request. If everything's alright, I will accept the request.</li>
</ol>
